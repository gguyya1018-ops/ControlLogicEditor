# 플랜트 도면 데이터 변환 및 활용 보고서

**작성일**: 2026-02-05

---

## 1. 개요

플랜트 운영에 필수적인 두 종류의 도면을 디지털 데이터로 변환하여 LLM 기반 검색 및 분석 시스템 구축

| 도면 종류 | 원본 형식 | 용도 |
|-----------|-----------|------|
| 제어 로직 | PDF | 자동화 제어 논리, 알람 원인 추적 |
| P&ID (배관계장도) | DWG (CAD) | 설비 위치, 배관 연결 확인 |

---

## 2. 데이터 변환 방법

### 제어 로직 도면 (PDF)

```
PDF → 이미지 변환 → OpenCV 이미지 분석 → OCR 텍스트 인식 → JSON
```

- **OpenCV**: 이미지에서 블록 위치를 자동 검출하는 컴퓨터 비전 라이브러리
- **OCR**: 이미지 내 텍스트를 인식하여 태그명, 설명 추출
- **좌표 출처**: 이미지 픽셀 분석 (원본에 좌표 데이터 없음)

### P&ID 도면 (DWG)

```
DWG → DXF 변환 → ezdxf 파싱 → JSON
```

- **DXF**: CAD 도면의 텍스트 기반 포맷 (좌표, 속성 데이터 포함)
- **좌표 출처**: CAD 원본 데이터 (정확한 수치 좌표)

## 4. 활용 가능 기능

| 기능 | 효과 |
|------|------|
| 태그 통합 검색 | 3,000+ 태그 즉시 검색 (3초 이내) |
| 신호 연결 추적 | 알람 발생 시 원인 로직 역추적 |
| 크로스 레퍼런스 | 하나의 태그가 등장하는 모든 도면 확인 |
| 설비 위치 확인 | P&ID에서 밸브/계기 물리적 위치 표시 |
| 정비이력 연동 | 설비 ID 기반 과거 정비 기록 조회 (연동 시) |

### 탐색 시간 단축 효과

| 단계 | 기존 방식 | 개선 후 |
|------|-----------|---------|
| 도면 찾기 | 30분 | 3초 |
| 연결 추적 | 15분 | 10초 |
| 이력 조회 | 10분 | 10초 |
| **총합** | **~55분** | **~30초** |

## 5. 한계점

### 가능한 것
- 연결고리 확인: "이 알람이 어떤 로직에서 발생했는가?"
- 위치 확인: "이 밸브가 P&ID 어디에 있는가?"
- 이력 조회: "이 설비의 과거 정비 기록은?"

### 불가능한 것
- 근본 원인 분석: "왜 진동이 발생했는가?"
- 고장 예측: "언제 고장날 것인가?"

**이유**: 진동/고장 원인 분석에는 로직 연결 정보가 아닌 다음이 필요
- 설비 스펙 (RPM, 베어링 타입 등)
- 센서 트렌드 데이터
- 고장 패턴 지식베이스

---

## 6. 향후 확장 방안

진단 기능 구현을 위해 필요한 추가 데이터:

| 데이터 | 용도 |
|--------|------|
| 설비 마스터 DB | 설비별 스펙, 구성품 정보 |
| 센서 트렌드 데이터 | 진동, 온도, 압력 시계열 |
| 고장 패턴 DB | 유형별 고장 특성, 징후 |

위 데이터가 연동되면 LLM 기반 고장 진단 가능:
> "현재 진동 패턴이 베어링 결함 초기 증상과 85% 유사합니다"

## 8. 결론

본 시스템은 **"진단 도구"가 아닌 "탐색 도구"**로서의 가치를 제공

- **즉시 효과**: 도면 탐색 시간 55분 → 30초 (99% 단축)
- **LLM 활용**: 자연어로 태그/설비 검색 가능
- **확장 가능**: 정비이력, 센서 데이터 연동 시 진단 기능으로 발전 가능

## 1. 목표: LLM을 활용한 로직 변경 지원

### 핵심 아이디어

```
제어 로직 도면 (그래픽)
        ↓
    구조화된 JSON 데이터
        ↓
    텍스트로 변환 (연결 정보)
        ↓
    LLM에 전달 (필요한 부분만)
        ↓
    "이 로직을 이렇게 변경하면 어떻게 될까?" 질문 가능
```

**기대 효과**:
- 블록 간 연결 관계를 텍스트로 표현 → LLM이 로직 구조를 이해
- 특정 태그의 입력/출력 신호 추적
- 로직 변경 시 영향 범위 분석 지원
- "이 밸브를 닫으면 어떤 신호가 영향받나?" 같은 질문에 답변 가능

## 2. 제어 로직 도면 vs P&ID 도면

### 개요

| 구분 | 제어 로직 도면 | P&ID (배관계장도) |
|------|---------------|------------------|
| **목적** | 자동화 제어 논리 표현 | 배관/계기 물리적 배치 |
| **내용** | 신호 흐름, 연산 블록, 인터락 조건 | 배관 라인, 밸브, 계기 위치 |
| **원본 형식** | PDF (Emerson Ovation 출력) | DWG/DXF (CAD 도면) |
| **활용** | "왜 이 밸브가 열렸나?" 추적 | "이 밸브가 어디 있나?" 확인 |

### 제어 로직 도면
- Emerson Ovation 시스템에서 전용 프로그램으로 PDF 출력
- 신호의 입력 → 연산 → 출력 흐름을 시각적으로 표현
- 블록 타입: OCB(제어블록), SIGNAL(신호), ALG(연산), REF(참조), PORT(입출력)

### P&ID 도면 (보조 기능)
- CAD 소프트웨어(AutoCAD)로 작성된 DWG 파일
- 플랜트 배관, 밸브, 계기의 물리적 연결과 위치
- 실제 현장에서 설비를 찾을 때 사용
- **주요 활용**: 태그 검색 시 물리적 위치(그리드 좌표) 빠른 확인

### 3.1 제어 로직 (PDF → JSON)

```

│                    원본: PDF 파일                            │
│            (Emerson Ovation 프로그램 출력물)                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              Step 1: 이미지 변환 (pdf2image)                 │
│                                                              │
│   PDF 각 페이지 → PNG 이미지 (300 DPI)                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              Step 2: 텍스트 추출 (PyMuPDF)                   │
│                                                              │
│   • PDF 내장 텍스트 + 좌표 정보 추출                          │
│   • 블록명, 태그명, 포트명 등 텍스트 데이터                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              Step 3: 이미지 분석 (OpenCV)                    │
│                                                              │
│   • 색상 필터링: 빨간색/녹색/노란색 블록 분리                  │
│   • 외곽선 검출: 사각형 블록 위치(x, y) 추출                  │
│   • 선 검출 (HoughLinesP): 연결선 경로 추적                   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              Step 4: 후처리 및 구조화                         │
│                                                              │

└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    최종: JSON 파일                           │
│                                                              │
│   {                                                          │
│     "blocks": [                                              │
│       { "id": "B001", "type": "OCB", "x": 120, "y": 340,    │
│         "text": "FCV-1551", "width": 80, "height": 60 }     │
│     ],                                                       │
│     "lines": [                                               │
│       { "x1": 200, "y1": 340, "x2": 300, "y2": 340 }        │
│     ],                                                       │
│     "groups": [...]                                          │
│   }                                                          │

```

**좌표 출처**: 이미지 픽셀 분석 (OpenCV)

## 4. OpenCV란?

**OpenCV** (Open Computer Vision) = 이미지/영상 처리 오픈소스 라이브러리

### 이 프로젝트에서 사용한 기능

| 기능 | 설명 | 용도 |
|------|------|------|
| 색상 필터링 | HSV 색공간에서 특정 색상 추출 | 빨간/녹색/노란 블록 분리 |
| 외곽선 검출 | contour detection | 사각형 블록 위치 찾기 |
| 선 검출 | Hough Line Transform | 연결선 경로 추적 |
| 이미지 전처리 | 이진화, 노이즈 제거 | 블록/선 검출 정확도 향상 |

### 예시 코드
```python
import cv2

# 이미지 로드
img = cv2.imread('logic_page.png')

# HSV 변환 후 빨간색 필터링
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
red_mask = cv2.inRange(hsv, (0, 100, 100), (10, 255, 255))

# 외곽선 검출
contours, _ = cv2.findContours(red_mask, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

# 각 외곽선의 바운딩 박스 (블록 위치)
for cnt in contours:
    x, y, w, h = cv2.boundingRect(cnt)
    print(f"블록 위치: ({x}, {y}), 크기: {w}x{h}")
```

## 5. 추출된 데이터 활용

### 5.1 LLM 활용 시나리오 (핵심 목표)

```
예시: "FCV-1551 밸브의 개도 로직을 변경하고 싶다"

1. 사용자가 태그 "FCV-1551" 검색
        ↓
2. 해당 태그의 연결 정보를 텍스트로 변환:
   "FCV-1551은 OCB_BLOCK 타입이며,
    입력: PID-OUT(자동 출력), MAN-SP(수동 설정값)
    출력: AO-1551(아날로그 출력)
    연결된 블록: M/A 스위치, PID 컨트롤러"
        ↓
3. LLM에 질문:
   "이 밸브의 개도 상한을 80%로 제한하려면
    어떤 블록을 추가해야 할까?"
        ↓
4. LLM 답변:
   "M/A 스위치와 AO 출력 사이에 HIGH SELECT 블록을
    추가하여 최대값을 제한할 수 있습니다.
    또는 PID 블록의 출력 제한 파라미터를 조정..."
```

### 5.2 기본 기능

| 기능 | 설명 | 예시 |
|------|------|------|
| **연결 추적** | 신호 흐름 역추적 | "이 알람이 어떤 로직에서 왔나?" |
| **태그 검색** | 5,900+ 태그 즉시 검색 | "FCV-1551" 검색 → 위치 표시 |
| **크로스 레퍼런스** | 하나의 태그가 여러 도면에 등장 | 태그 클릭 → 관련 도면 목록 |
| **설비 위치** | P&ID에서 물리적 위치 확인 | 밸브 ID → 그리드 좌표 "5A" |

## 6. LLM 활용 가능 범위

### 6.1 가능한 것: 로직 구조 이해 및 변경 지원

```
블록 연결 정보를 텍스트화하면 LLM이 이해 가능:

"OCB008802C (T 블록)
 - 입력: FLAG (조건 신호)
 - 출력: YES (참일 때), NO (거짓일 때), OUT
 - 연결: ALG-0113015 (SUB 블록)에서 신호 수신"

이런 형태로 전달하면 LLM이 로직 구조를 파악하고
변경 시 영향 범위, 추가할 블록 등을 제안할 수 있음
```

**LLM이 도울 수 있는 질문들:**
- "이 밸브의 인터락 조건을 추가하려면?"
- "A 신호가 B에 영향을 주는 경로는?"
- "이 로직을 이렇게 바꾸면 어떤 문제가 생길 수 있나?"
- "PID 출력에 제한을 걸려면 어떤 블록을 써야 하나?"

### 6.2 불가능한 것: 물리적 현상 진단

```
"왜 진동이 발생했는가?" ← LLM으로 불가능
```

**이유**: 로직 연결 정보만으로는 물리적 현상(베어링 마모, 캐비테이션 등)을 진단할 수 없음

### 6.3 핵심 가치

| 기존 방식 | 개선된 방식 |
|-----------|-------------|
| PDF 뒤지며 연결 추적 (30분+) | 태그 검색 → 연결 정보 텍스트화 (10초) |
| 경험에 의존한 로직 변경 | LLM에게 변경 방안 자문 가능 |
| 영향 범위 수동 추적 | 연결된 블록 목록 자동 제공 |
